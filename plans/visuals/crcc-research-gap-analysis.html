01 — Current State of the Field

What exists at the noisy labels × class imbalance intersection

Detection Layer

Confident Learning

Northcutt et al. (2021) — estimates noise via joint distribution of noisy/true labels. Works well on balanced data. Detects but does not constrain which flagged samples to delete.

Detection: strong Intervention: silent

Detection Layer

Loss-Based Scoring (CTRL, Step-E)

Small-loss prior: high loss → likely noisy. Fatal flaw under imbalance — minority samples are harder to learn, so clean minority instances score high. No per-class safeguard.

Biased under imbalance

Deep Learning Methods

Class-Aware (Liu 2024, Sheng 2024)

Liu: class-balance regularization inside training. Sheng: adaptive sample selection + semi-supervised training. Both model-coupled — require modifying the training loop. Not post-hoc.

DL-only, not post-hoc

Deep Learning Methods

UNICON / DivideMix / Co-teaching

Karim 2022, Li 2020, Han 2018. Contrastive learning, GMM selection, disagreement-based filtering. All require deep network representation learning — impractical for tabular gradient boosting / logistic regression.

Not tabular-applicable

Active Cleaning

Active Label Cleaning

Bernhardt (2022), Khanal MICCAI (2024). Focus on budgeted relabeling with human annotators. Different intervention type — not automated deletion. Per-class budget allocation is not studied.

Relabeling, not deletion

Tabular-Specific

Tabular Noisy Label Work

Salekshahrezaee (2021), TabMentor (2023). Focus on detection on tabular data via reconstruction error. No guidance on post-detection intervention for imbalanced tables.

Detection only

02 — Overarching Gaps

What the field systematically lacks

Primary Gap

Post-detection intervention rules for imbalanced tabular ML. Once samples are scored as suspicious, the question of which to delete under a budget, constrained by class safety, has no practical answer in the literature. Every existing class-aware method addresses this inside the training loop — not as a clean post-hoc step.

Gap 1

No per-class deletion caps

Literature mentions class imbalance as a detection challenge. Nobody proposes a hard constraint on maximum deletions per class as a post-processing rule. Active cleaning papers use labeling budgets, not deletion budgets.

Gap 2

No CMDR metric

Clean-minority deletion rate — fraction of deleted samples that are correctly-labeled minority instances — doesn't appear in any reviewed paper. Standard metrics (accuracy, F1) are blind to this collateral damage. You can improve accuracy while gutting minority recall.

Gap 3

Model-agnostic tabular rule

Tabular practitioners use logistic regression and gradient boosting. Existing class-aware methods assume deep networks with representation learning, batch normalization, and 100+ epoch training. There is no plug-in post-hoc rule for classical ML pipelines.

Gap 4

When does risk-adjustment matter?

Even assuming caps exist, the question of when class-risk-adjusted scoring (lambda) adds value over plain class-proportional caps is uncharted. Our data shows it doesn't matter under extreme imbalance — but the regime where it does is unexplored.

CRCC Claim

The defensible novelty

Prior work has studied class-aware noisy-label sample selection, mostly inside integrated deep-learning pipelines. CRCC studies a model-agnostic post-detection intervention rule for imbalanced tabular ML — with a new harm metric (CMDR) and an ablation isolating caps vs. risk-adjusted scoring.

Key Finding (Surprising)

Lambda is dead weight under extreme imbalance

Under 85/15 imbalance, the proportional cap is hit so quickly (6–12 samples) that adjusted-score ranking never influences the selection. Lambda across {0, 0.25, 0.5, 1.0} produces identical results — the cap dominates before any cross-class competition occurs.

This is a finding, not a failure

Cap design is the primary lever for extreme imbalance. Lambda is the lever for moderate imbalance + larger budgets — a regime not yet tested.

What's Confirmed
  • Under 85/15 imbalance: cap = binding constraint, lambda = no effect
  • Class-proportional cap ≡ CRCC-P under extreme imbalance (lambda insensitive)
  • CRCC-M (halved minority cap) further reduces CMDR at comparable accuracy
  • Effect is consistent across LR and HGB — model-agnostic confirmed
  • Sick (6.1% minority) shows near-perfect CMDR protection for all cap methods
What's Still Open
  • Does lambda matter under moderate imbalance (e.g. 70/30)?
  • Does a larger budget (20%) push lambda into relevance?
  • What is the threshold imbalance ratio where caps become non-binding?
  • Real-world noise (not synthetic) — CMDR can't be measured without ground truth
  • Multi-class extension (currently binary only)
06 — Final Experiment

The mild-imbalance experiment: finding lambda's regime

Why this closes the story

The main experiment proves caps work. The journal explicitly flags: "Consider running a second experiment with milder imbalance (e.g., 70/30) and larger budget (20%) to find the regime where lambda matters." Without this, the risk-adjusted scoring component of CRCC remains unvalidated in any regime — reviewers will ask.

Experiment Design
  • Imbalance ratio: 70/30 (instead of 85/15)
  • Deletion budget: 20% (instead of 10%)
  • Same 3 datasets, 2 models, 5 seeds
  • Same λ grid: {0, 0.25, 0.5, 1.0}
  • Key metric: do different λ values produce different CMDR / recall?
  • Expected scale: ~120 rows (3×2×5×4 lambda variants)
Expected Outcomes
  • If lambda still no effect: caps dominate even at 70/30. Conclusion: cap design is the only lever for imbalance levels practitioners encounter.
  • If lambda matters at 70/30: confirms the risk-adjusted scoring component adds value in the moderate-imbalance regime. Strengthens the CRCC-P novelty claim.
  • Either outcome is publishable — one tightens the claim, one extends it.
